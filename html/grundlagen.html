Kapitel 1: Grundlagen der ABAP-Programmierung

Lektion 1: Entwickeln eines einfachen ABAP-Programms

Ein ABAP-Programm wird im ABAP Editor (z.B. Transaktion SE38 oder SE80) angelegt. Jedes Programm benötigt Attribute, wie den Typ (z.B. "Ausführbares Programm") und einen Titel. Das erste Schlüsselwort in einem Report ist oft REPORT. Die Ausgabe erfolgt klassisch mit WRITE. Jede ABAP-Anweisung endet mit einem Punkt. Wichtige Funktionen sind Sichern ($STRG+S$), Syntax prüfen ($STRG+F2$) und Aktivieren ($STRG+F3$).

Lektion 2: Einführung in die ABAP-Syntax

ABAP ist eine plattformunabhängige , aufwärtskompatible Sprache, die SQL-Zugriffe und Objektorientierung unterstützt.

  • Anweisungen: Beginnen mit einem Schlüsselwort und enden mit einem Punkt. Wörter werden durch Leerzeichen getrennt.
  • Kettensätze: Fassen Anweisungen mit gleichem Schlüsselwort zusammen. Sie beginnen mit einem Doppelpunkt (:) nach dem Schlüsselwort , und die einzelnen Anweisungsteile werden durch Komma (,) getrennt.
  • Kommentare: Ein Stern (*) am Zeilenanfang kommentiert die ganze Zeile aus. Ein Anführungszeichen (") kommentiert den Rest der Zeile aus.
  • Hilfe: Die F1-Taste auf einem Schlüsselwort öffnet die ABAP-Schlüsselwortdokumentation.

Lektion 3: Implementieren von einfachen Dialogen

Um Benutzereingaben zu ermöglichen, wird ein Selektionsbild verwendet. Das Schlüsselwort PARAMETERS deklariert eine Variable für die Dateneingabe. Diese Variable (z.B. pa_name) fungiert als Datencontainer , dessen Inhalt im Programm für die Verarbeitung (z.B. WRITE pa_name) genutzt werden kann.

Lektion 4: Anpassen des ABAP Editors

Es gibt den klassischen und den neuen ABAP Editor. Der neue Editor bietet Syntax-Hervorhebung , Zeilennummern und Code-Vorlagen. Der Pretty Printer formatiert den Quelltext automatisch (z.B. Einrückung und Groß-/Kleinschreibung).

Kapitel 2: Quelltext schreiben und Debuggen in ABAP

Lektion 1: Definieren von einfachen Variablen

Variablen werden benötigt, um Daten im Programm zu speichern. ABAP unterscheidet elementare Standardtypen und globale Typen aus dem ABAP Dictionary.

  • Standardtypen: I (Integer) , P (Gepackte Zahl, für Dezimalzahlen) , STRING (Zeichenkette variabel) , C (Zeichenkette fest) , D (Datum) , T (Uhrzeit).
  • Dictionary-Typen: Man kann sich auf bestehende Typen beziehen, z.B. TYPE MARA-MATNR.
  • Deklaration: Erfolgt mit DATA. Beispiel: DATA gv_var1 TYPE i.. Mit LIKE kann man sich auf eine andere Variable beziehen: DATA gv_var2 LIKE gv_var1..
  • Literale: Feste Werte im Code. Textliterale stehen in Hochkommata ('Hello World') , Zahlenliterale nicht (123).
  • Konstanten: Variablen mit festem Wert, die nicht geändert werden können. Deklaration: CONSTANTS c_hello TYPE string VALUE 'Hello World'..

Lektion 2: Definieren von Textsymbolen

Um Programme mehrsprachig zu gestalten, werden Textliterale vermieden. Stattdessen verwendet man Textsymbole. Im Code wird ein Textsymbol über eine ID (z.B. 'hel') referenziert: WRITE: 'Hello' (hel), pa_name.. Die Übersetzungen (DE, EN, FR etc.) werden in den Textelementen des Programms gepflegt.

Lektion 3: Durchführen von arithmetischen Operationen

Variablen haben je nach Typ einen Initialwert (z.B. I hat 0, STRING ist leer). Werte können zugewiesen werden (gv_result = pa_num1 + pa_num2.). Die gängigen Operatoren sind +, -, *, /. Alternativ gibt es Schlüsselwörter wie ADD 1 TO gv_result..

Lektion 4: Verwenden von Systemvariablen

Das System stellt vordefinierte Variablen (Systemfelder) bereit, die häufig genutzte Informationen enthalten.

  • SY-DATUM: Aktuelles Datum des Applikationsservers.
  • SY-UZEIT: Aktuelle Uhrzeit des Applikationsservers.
  • SY-UNAME: Benutzer-ID des angemeldeten Benutzers.

Lektion 5: Debuggen von Programmen

Der ABAP Debugger ist das Werkzeug zur Fehleranalyse. Man startet ihn, indem man /h in das Befehlsfeld eingibt und Enter drückt , oder indem man Breakpoints im Code setzt. Im Debugger kann man den Wert von Variablen prüfen (per Doppelklick) und den Code schrittweise ausführen (z.B. Einzelschritt mit F5). Watchpoints können gesetzt werden, um das Programm anzuhalten, sobald sich der Wert einer bestimmten Variable ändert.

Lektion 6: Anlegen von ABAP-Listen

Klassische ABAP-Listen sind die Standardausgabe von Reports (erzeugt durch WRITE). Sie bieten Standardfunktionen wie Drucken, Suchen und Sichern. Die Ausgabe kann formatiert werden, z.B. mit Farben (WRITE ... COLOR COL_POSITIVE.) oder Ikonen (WRITE ... AS ICON.). Listen- und Spaltenüberschriften können über die Textelemente gepflegt werden.

Lektion 7: Zeichenkettenverarbeitung

ABAP bietet Befehle zur Manipulation von Zeichenketten:

  • Verketten: CONCATENATE pa_fname pa_lname INTO gv_full_name SEPARATED BY space..
  • Suchen: FIND 'Smith' IN pa_name.. Das Ergebnis (Erfolg/Misserfolg) wird in SY-SUBRC gespeichert.
  • Teilen: SPLIT gv_text AT ' ' INTO gv_fname gv_lname..

Kapitel 3: Ablaufsteuerungsstrukturen in ABAP

Lektion 1: Implementieren von Bedingungslogik

Bedingungslogik (bedingte Verzweigungen ) steuert, ob ein Codeblock ausgeführt wird.

  • Struktur: Die Anweisung beginnt mit IF und endet mit ENDIF..
  • Alternativen: Mit ELSE wird ein alternativer Block ausgeführt, falls die IF-Bedingung nicht zutrifft. Mit ELSEIF können weitere Bedingungen geprüft werden.
  • Vergleichsoperatoren: = (EQ), <> (NE), > (GT), < (LT), >= (GE), <= (LE).
  • Boolesche Operatoren: Bedingungen können mit AND und OR verknüpft werden. NOT negiert eine Bedingung.

Lektion 2: Implementieren von Schleifen

Schleifen werden für sich wiederholende Logik verwendet. Der aktuelle Schleifendurchlauf wird in der Systemvariablen SY-INDEX gezählt.

  • Endlosschleife (mit Abbruch): DO. ... IF . EXIT. ENDIF. ... ENDDO..
  • Zählschleife: DO n TIMES. ... ENDDO.. Führt den Block n-mal aus.
  • Bedingte Schleife: WHILE . ... ENDWHILE.. Prüft die Bedingung vor jedem Durchlauf.

Kapitel 4: Laufzeitfehler und Fehlerbehandlung

Lektion 1: Analysieren von Laufzeitfehlern

Auch syntaktisch korrekte Programme können zu Laufzeitfehlern führen, z.B. bei einer Division durch Null. Dies führt zu einem ABAP-Kurzdump (analysierbar in Transaktion ST22). Der Dump zeigt den Fehlernamen (z.B. BCD_ZERODIVIDE) und den Quelltextauszug an der Fehlerstelle. Solche Fehler können durch präventive Bedingungslogik (z.B. IF pa_mths <> 0.) vermieden werden.

Lektion 2: Implementieren von Fehlerbehandlung

Statt das Programm abstürzen zu lassen, sollte eine kontrollierte Fehlerbehandlung erfolgen, oft durch das Senden von Nachrichten.

  • Nachrichtentypen: I (Information) stoppt das Programm und zeigt ein Dialogfenster. E (Fehler) stoppt die Verarbeitung und zeigt die Nachricht in der Statuszeile an.
  • Nachrichtenklassen: Nachrichten werden in Nachrichtenklassen (Transaktion SE91) verwaltet.
  • Aufruf: MESSAGE i001(bc100_msg).. Um Variablen in den Text zu integrieren (Platzhalter) wird der Zusatz WITH verwendet: MESSAGE i001(bc100_msg) WITH pa_hols..

Kapitel 5: Zusätzliche ABAP-Programmiertechniken

Lektion 1: Abrufen von Daten aus der Datenbank

Der Zugriff auf Datenbanktabellen (definiert im ABAP Dictionary, SE11 ) erfolgt über die Datenbankschnittstelle mittels Open SQL.

  • SELECT-Anweisung: Der zentrale Befehl zum Lesen von Daten.
  • Einzelnen Satz lesen: SELECT SINGLE ... FROM ... INTO ... WHERE ....
    • SELECT: Welche Spalten?
    • FROM: Welche Tabelle?
    • INTO: Wohin im Programm? (z.B. eine Struktur)
    • WHERE: Welche Zeilen? (Filterbedingung)
  • Erfolgsprüfung: Nach einem SELECT-Befehl sollte SY-SUBRC geprüft werden. Wenn SY-SUBRC = 0, war der Zugriff erfolgreich.

Lektion 2: Modularisierung in ABAP

Modularisierung dient der Wiederverwendung von Quelltext. Statt denselben Codeblock in viele Programme zu kopieren, wird er zentral gekapselt und aufgerufen. Der Datenaustausch zwischen Aufrufer und aufgerufenem Block darf nicht über globale Variablen erfolgen , sondern muss über eine definierte Schnittstelle geschehen.

Lektion 3: Verwenden von Funktionsbausteinen

Funktionsbausteine (Transaktion SE37 ) sind eine wichtige Technik zur Modularisierung. Sie besitzen eine feste Schnittstelle:

  • Importparameter: Daten, die an den Baustein übergeben werden (Eingabe).
  • Exportparameter: Daten, die vom Baustein zurückgegeben werden (Ausgabe).
  • Changing-Parameter: Werden übergeben und verändert zurückgegeben.
  • Ausnahmen (Exceptions): Dienen der Fehlerbehandlung. Tritt eine Ausnahme auf, wird SY-SUBRC auf einen Wert ungleich 0 gesetzt.

Der Aufruf im Programm erfolgt mit CALL FUNCTION 'FUNKTIONSBAUSTEIN_NAME' .... Im ABAP Editor kann die "Pattern"-Drucktaste (oder $STRG+F6$) verwendet werden, um eine Vorlage für den Aufruf zu generieren.
