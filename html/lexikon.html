ABAP Lexikon

Eine Sammlung der wichtigsten Begriffe, die auf dieser Webseite erwähnt werden.

A

ABAP (Advanced Business Application Programming)
Die primäre Programmiersprache für Geschäftsanwendungen im SAP-Umfeld. Sie ist ereignisgesteuert und wird auf dem ABAP Application Server ausgeführt.
ABAP Dictionary (SE11)
Zentrales Werkzeug zur Definition und Verwaltung von datenbankunabhängigen Datentypen und Datenbankobjekten. Hier werden Tabellen, Views, Datenelemente, Domänen etc. angelegt.
ABAP Editor (SE38)
Die Entwicklungsumgebung zum Schreiben, Testen und Ausführen von ABAP-Programmen (Reports).
ADT (ABAP Development Tools)
Eine moderne, auf Eclipse basierende Entwicklungsumgebung für ABAP, die Features wie Quick Fixes, Refactoring und integrierte Testwerkzeuge bietet.
ALV (ABAP List Viewer)
Ein mächtiges Werkzeug zur Darstellung von tabellarischen Daten in Listenform. Bietet Standardfunktionen wie Sortieren, Filtern, Summenbildung und Layout-Verwaltung. Die moderne Variante ist CL_SALV_TABLE.
Anwendungsschicht (Application Layer)
Die mittlere Schicht in der 3-Schichten-Architektur, auf der die ABAP-Anwendungslogik im Application Server ausgeführt wird.

B

Breakpoint
Ein Haltepunkt im Quellcode, der die Ausführung des Programms im Debugger an dieser Stelle unterbricht, um eine Analyse zu ermöglichen.

C

CDS Views (Core Data Services)
Eine moderne Technologie zur Definition von semantischen Datenmodellen direkt auf der Datenbank (z.B. SAP HANA). Sie erweitern SQL um Annotationen und ermöglichen "Code Pushdown".
Client (Mandant)
Eine organisatorisch und technisch eigenständige Einheit innerhalb eines SAP-Systems mit eigenen Stammdaten und Bewegungsdaten. Die Anmeldung erfolgt immer an einem bestimmten Mandanten.
CONCATENATE
ABAP-Anweisung zur Verkettung von mehreren Zeichenketten.

D

Datenelement
Ein Objekt im ABAP Dictionary, das die semantische Bedeutung eines Feldes beschreibt (z.B. Feldbezeichner, F1-Hilfe). Es referenziert eine Domäne.
Datenbankschicht (Database Layer)
Die unterste Schicht der 3-Schichten-Architektur, die für die physische Speicherung der Daten zuständig ist (z.B. eine SAP HANA Datenbank).
Debugger
Ein Werkzeug zur schrittweisen Analyse eines Programms zur Laufzeit. Er wird oft mit /h im Befehlsfeld gestartet.
Domäne
Ein Objekt im ABAP Dictionary, das die technischen Eigenschaften eines Feldes definiert (Datentyp, Länge, Wertebereich).
DO-Schleife
Eine Schleifenstruktur in ABAP, die entweder eine feste Anzahl von Durchläufen (DO n TIMES) oder eine Endlosschleife (die mit EXIT verlassen werden muss) implementiert.

E

Exception (Ausnahme)
Ein Mechanismus zur Fehlerbehandlung. In Funktionsbausteinen oder Methoden können Ausnahmen definiert werden, die im Fehlerfall ausgelöst und vom aufrufenden Programm behandelt werden können.

F

Funktionsbaustein (Function Module)
Eine wiederverwendbare, prozedurale Code-Einheit mit einer definierten Schnittstelle (Import, Export, Changing, Tables, Exceptions). Wird in der Transaktion SE37 gepflegt.

H

Hash-Tabelle
Ein Typ einer internen Tabelle, der für extrem schnelle Lesezugriffe über einen eindeutigen Schlüssel optimiert ist. Der Zugriff erfolgt über einen Hash-Algorithmus.

I

IF-Anweisung
Eine grundlegende Ablaufsteuerungsstruktur für bedingte Logik (Verzweigungen). Besteht aus IF, optional ELSEIF und ELSE, und wird mit ENDIF abgeschlossen.
Include
Ein ABAP-Programmtyp, der dazu dient, Quellcode zu strukturieren, indem er in andere Programme eingebunden wird. Er kann nicht eigenständig ausgeführt werden.
Interne Tabelle
Ein zentrales Konstrukt in ABAP zur Haltung und Verarbeitung von tabellarischen Daten im Arbeitsspeicher eines Programms. Es gibt die Typen Standard, Sortiert und Hash.

J

JOIN
Eine Operation in Open SQL, um Daten aus zwei oder mehr Datenbanktabellen basierend auf einer Verknüpfungsbedingung zu kombinieren.

K

Kettensatz
Eine syntaktische Vereinfachung in ABAP, um mehrere Anweisungen mit demselben einleitenden Schlüsselwort zusammenzufassen. Beginnt mit einem Doppelpunkt (:) und trennt die Teile mit Kommas (,).
Klasse
Das zentrale Konstrukt der objektorientierten Programmierung in ABAP. Sie kapselt Daten (Attribute) und Verhalten (Methoden). Wird im Class Builder (SE24) oder lokal im Programm definiert.
Kurzdump
Ein Laufzeitfehler, der die Ausführung eines Programms abbricht. Kann in der Transaktion ST22 analysiert werden.

L

Literal
Ein fester Wert im Quellcode. Textliterale werden in Hochkommata ('...') geschrieben, Zahlenliterale direkt.

M

Methode
Eine Funktion oder Prozedur, die zu einer Klasse gehört und deren Verhalten implementiert.
Modularisierung
Das Aufteilen von Code in wiederverwendbare, gekapselte Einheiten wie Unterprogramme, Funktionsbausteine oder Methoden, um Redundanz zu vermeiden und die Wartbarkeit zu verbessern.

O

Open SQL
Die datenbankunabhängige Datenbanksprache von ABAP. Anweisungen wie SELECT, INSERT, UPDATE, DELETE werden vom ABAP Application Server in das native SQL der jeweiligen Datenbank übersetzt.

P

Paket (Package)
Ein Container für zusammengehörige Entwicklungsobjekte. Pakete steuern die Transportierbarkeit von Objekten. Objekte im lokalen Paket $TMP sind nicht transportierbar.
PARAMETERS
ABAP-Schlüsselwort zur Definition eines einzelnen Eingabefeldes auf einem Selektionsbild.
Präsentationsschicht (Presentation Layer)
Die oberste Schicht der 3-Schichten-Architektur, die die Benutzeroberfläche darstellt (z.B. SAP GUI oder ein Webbrowser mit Fiori).
Pretty Printer
Ein Werkzeug im ABAP Editor, das den Quellcode automatisch formatiert (z.B. Einrückungen, Groß-/Kleinschreibung), um die Lesbarkeit zu verbessern.

R

RFC (Remote Function Call)
Ein Protokoll, das den Aufruf eines Funktionsbausteins in einem anderen SAP-System oder von einem externen System aus ermöglicht.

S

SAP GUI
Die grafische Benutzeroberfläche für den Zugriff auf SAP-Systeme. Sie ist die klassische Präsentationsschicht.
SE80 (Object Navigator)
Die zentrale Transaktion der klassischen ABAP Workbench. Sie bietet eine baumartige Übersicht über alle Entwicklungsobjekte in einem Paket.
Selektionsbild (Selection Screen)
Die standardisierte Eingabemaske eines ausführbaren ABAP-Programms, die mit PARAMETERS und SELECT-OPTIONS definiert wird.
SELECT
Die zentrale Open-SQL-Anweisung zum Lesen von Daten aus Datenbanktabellen.
SELECT-OPTIONS
ABAP-Schlüsselwort zur Definition eines komplexen Eingabebereichs auf einem Selektionsbild, der Einzelwerte, Intervalle und Muster erlaubt.
Sortierte Tabelle
Ein Typ einer internen Tabelle, deren Einträge immer nach ihrem Schlüssel sortiert sind. Dies ermöglicht schnelle Lesezugriffe über binäre Suche.
Standardtabelle
Der flexibelste Typ einer internen Tabelle. Der Zugriff erfolgt über einen Index oder linear über den Schlüssel. Sie hat keinen fest definierten Schlüssel.
Systemfeld
Eine vordefinierte globale Variable, die zur Laufzeit vom System mit Informationen versorgt wird (z.B. SY-SUBRC, SY-DATUM, SY-INDEX).
SY-SUBRC
Ein wichtiges Systemfeld, das den Rückgabewert (Erfolgscode) einer Anweisung anzeigt. Ein Wert von 0 bedeutet in der Regel "erfolgreich", ein Wert ungleich 0 signalisiert einen Fehler oder eine besondere Bedingung.

T

Transaktion
Ein Kürzel (Transaktionscode), das eine bestimmte Funktion oder ein Programm im SAP-System aufruft (z.B. SE80, ST22).
Transport
Ein Mechanismus, um Entwicklungsobjekte (z.B. Programme, Tabellen) von einem System in ein anderes zu übertragen (z.B. vom Entwicklungs- ins Testsystem).

V

Variable
Ein benannter Datencontainer im Programm, der zur Laufzeit Werte speichern kann. Wird in ABAP mit DATA deklariert.

W

Watchpoint
Ein spezieller Haltepunkt im Debugger, der die Programmausführung stoppt, sobald sich der Wert einer bestimmten Variable ändert.
WHILE-Schleife
Eine Schleifenstruktur, die den Schleifenkörper so lange ausführt, wie eine angegebene Bedingung wahr ist. Die Bedingung wird vor jedem Durchlauf geprüft.
WRITE
Die klassische ABAP-Anweisung zur Ausgabe von Daten in einer Liste.
